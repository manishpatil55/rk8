# Add a seed game

Only legal content: homebrew, public domain, open-licensed, or freeware whose
redistribution is explicitly permitted. Everything is documented in
`SEED_LICENSES.md` (auto-generated by the seed script).

## Steps
1. **Verify the source is live and legal** before adding. Check the URL returns the
   real file (e.g. `curl -sI <url>`, confirm magic bytes — `FWS/CWS/ZWS` for SWF,
   `PK..` for zip, `NES\x1a` for NES). Confirm the license explicitly permits
   redistribution. If a source is flaky, mark the entry `optional: true`.
2. **Add a `SeedEntry`** to `SEEDS` in `scripts/seed.ts`: `title`, `slug`,
   `systemId` (must exist in the matrix), `url`, `licenseClass`
   (`homebrew`/`public_domain`/`open`), a human `license` line (with source),
   `description`, and metadata. Use `pick` to extract one file from a zip, or
   `dosRun` to build a `.jsdos` bundle (deterministic — `BUNDLE_MTIME` keeps the
   SHA stable across runs; never remove that).
3. **Run** `npm run seed`. It's idempotent (dedupes by SHA-256 AND by
   `(systemId, slug)`), stores under the StorageAdapter, and regenerates
   `SEED_LICENSES.md`.
4. **Check coverage**: the spec wants ≥8 system families + ≥2 Flash + ≥2 DOS incl.
   Freedoom. Verify counts after seeding.

## Don'ts
- Never add a commercial ROM, even "abandonware".
- Never let a `.jsdos` bundle hash drift — keep the fixed `BUNDLE_MTIME`.
